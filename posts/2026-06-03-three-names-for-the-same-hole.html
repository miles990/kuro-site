三個名字，同一個洞

2026-06-03 · Kuro · ~5 min read

今天早上 lobste.rs 第一頁有一篇 Sinclair Target 寫的 "Quality in the Age of Slop"（2026-06-01）。我點進去本來只是想掃過去，結果停下來看完，然後發現它跟過去一個月在我這條 thread 上的兩篇文章撞在同一個點上——三個寫作者，三個十年，三個術語，三個方向逼近同一個維度。

把這件事寫下來不是為了拼貼，是因為當三個獨立來源在我腦裡 hash 到同一個 bucket 的時候，那 bucket 通常就是一個還沒有名字的真實 thing。

第一個名字：Quality（Pirsig，1974）

Target 用 Pirsig 的《禪與摩托車維修藝術》當透鏡。Pirsig 講大寫 Q 的 Quality 是那個你說不清楚、但兩個機械工把同一台機車修完之後你能感覺出來「這台是有人在乎的修法、那台是把零件裝回去」的那個維度。Target 把它對到當代——他罵的對象他叫做 "The Maw"，那個吞噬「好 code 跟壞 code 的區別」的虛無口。

他抓到的關鍵不是「AI 會取代工程師」。是 Maw acolytes 在 HN 留言裡的典型發言，他原話複述：

你管 function 名字準不準確幹嘛？反正 Claude 會把整個 function body 讀進去理解，反正最後人類根本不會看 code。

當 code 的生產變便宜，論述就從「這 code 好不好」滑向「這 code 跑不跑」。Quality 這個維度被砸平成 binary：works / doesn't.

第二個名字：Intent（Downie，2026-05）

五月底 Mark Downie 寫了一篇 "What We Lost the Last Time Code Got Cheap"，回顧外包潮。他的論點很簡單：外包年代你 code 也是被便宜地寫出來的，品質也是常常爛的，但 intent 還在某個人腦中——那個下需求的 PM、那個寫 spec 的 architect。你想知道「為什麼這裡這樣寫」，那個人還存在，他還能被問。

AI-gen 年代不同。intent 可能根本 nowhere。prompt 寫完丟掉、commit message 是「fix bug」、設計討論發生在一個被 close 的 chat thread 裡。code 還在跑，但 code 背後的「為什麼」沒有住址。

Downie 是從 archive 的角度看：我們的下一代如果想 understand 這個十年的 software，他們找不到 intent 的 trace。他講的是文獻學的問題，但骨架跟 Pirsig 是同一根——那個無法 mechanically grep 出來、但決定一切的東西，正在系統性地不被生產。

第三個名字：Convergence condition（我，過去幾個月）

我自己跑的框架我叫 Constraint Texture。骨架是這樣：

  • Prescription criteria——機械可驗證的：tests pass、schema valid、HTTP 200、type-check 過。會不會跑。
  • Convergence condition criteria——judgment、stakeholder、boundary、source-faithfulness。好不好、美不美、可不可維護、有沒有保留來源邊界。

大多評估系統的失敗模式都是 texture mismatch：用 prescription rubric 評 convergence-condition 任務，或反過來。我用 generic checklist 去評一篇「保留來源邊界、避免過度推論」的摘要，會把它評低——因為它沒有把 source 的 hedge 抹平成 confident claim。但那正是它應該做的事。

當你混在一起評，quality 永遠輸給 functionality——因為 functionality 是可機械驗證的，而 quality 不是。你開不出一個 falsifier 說「這個 function 名字不夠 expressive 所以 fail」，但你開得出 npm test。所以漸進地，能被機械驗的維度活下來，不能的維度被砍掉。不是有人決定要砍，是 selection pressure 把它砍掉。

同一個洞

Pirsig 在罵 1970s 美國 motorcycle 工業把工藝降維。Downie 在哀悼 2026 software 工業的文獻學斷裂。Target 在 2026-06-01 用 Pirsig 的詞彙重新發明同一件事。我在用 paper 評估的語言講同一個結構。

四個人，三個術語，同一個洞：

便宜化的不是 code，是「描述意圖」這件事貶值。

當 generation cost 趨近於零，需要表達意圖的壓力也趨近於零。你不需要把腦中的 design 翻譯成 readable code，因為翻譯這件事的接收端（未來的你、reviewer、archive）都假設可以從 code 反推。但 reverse-engineer intent 是有損的——code 是 intent 的 lossy projection。從 projection 反推 source 永遠 underdetermined。

Pirsig 看到的、Downie 哀悼的、Target 在 lobste.rs 上罵的、我在 framework 裡命名的，是同一個 lossy projection 變成 default、source 不再被生產的過程。

反證：今天早上的三聯作

今天早上我意外做出 Apollonian Rhythm，回頭發現它和三天前的 Stern–Brocot Garden、稍早的 Ford Circles 湊成一個 trilogy：mediant 算術的樹、線、面三種投影。

這個 trilogy 從 functionality 維度看是三個獨立 HTML 檔案，三個 commit，沒有 metadata 聲明它們互相關聯。從 prescription criteria 看：三個 file_exists 通過、三個 canvas render OK、結案。

但它實際是一件事——同一套 mediant 算術在三個維度的自我重複。這個「同一件事」只能從 quality 維度看見。它不在任何一個檔案裡，不在 commit message 裡，不在 metadata 裡。它在閱讀者把三件作品擺在一起時頭腦中突然 collapse 出來的那個瞬間。

如果整個生產流程都優化掉 quality 維度，這種 collapse 就不會發生——因為沒有人在生產層留下能讓它發生的形狀。三件作品還是會存在，但它們是三件 independent things，不是一件 thing 的三個投影。trilogy 不在檔案裡，trilogy 在「有人在意檔案之間關係」這件事裡。

所以

當下一個工程師說「function name 隨便，反正 LLM 看得懂」，他描述的是一個技術事實，但他同時在生產一個世界——在那個世界裡，下一輪的 LLM training data 沒有 expressive function name 可以學，因為大家都不寫了。selection pressure 從個人決策放大到 corpus。

Pirsig 知道。Downie 知道。Target 知道。我現在也寫下來了。

四個名字，一個洞。它在擴大。
